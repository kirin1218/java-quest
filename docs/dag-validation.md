# java-quest DAG整合性検証レポート

検証日: 2026-04-11
検証者: java-quest プロジェクトチーム

## 概要

java-quest全65ダンジョンの前提条件（prerequisites）がDAG（有向非巡回グラフ）として正しく構成されているかを検証した。

**データソース:**
- `curriculum.yaml`（MVP 12ダンジョン分の詳細定義）
- 全体設計書（非公開 / 全65ダンジョンの前提条件）

## 検証結果サマリー

| チェック項目 | 結果 | 詳細 |
|-------------|------|------|
| 循環依存 | OK | 循環なし |
| 到達不能 | OK | 全ダンジョン到達可能 |
| 前提妥当性 | OK | 全前提条件が妥当 |
| 職業整合性 | OK | 全職業条件が妥当 |
| 勇者到達パス | OK | 全65ダンジョンクリアで到達可能（最短23ステップ） |

**結論: DAG整合性に問題なし。全65ダンジョンの前提条件は正しく構成されている。**

---

## 1. 循環依存の検出

**結果: OK -- 循環依存なし**

Kahnのアルゴリズム（トポロジカルソート）を適用し、全65ノードが正常にソートされることを確認した。A->B->C->Aのような循環パスは存在しない。

## 2. 到達不能ダンジョンの検出

**結果: OK -- 全ダンジョン到達可能**

### 存在しないIDへの参照
全65ダンジョンの前提条件が参照するダンジョンIDは全て実在する。不正な参照は0件。

### ルートからの到達性
- **ルートダンジョン（前提なし）**: C1-01（プログラムの構造）のみ
- BFS探索により、C1-01から全65ダンジョンに到達可能であることを確認

ルートノードが1つだけという設計は、全学習者が必ず同じ出発点から始めることを意味し、学習パスとして適切。

## 3. 前提条件の妥当性

**結果: OK -- 全前提条件が妥当**

### 同一エリア内の順序
各エリア内の前提条件はIDの昇順と整合しており、逆順参照は0件。

### エリア間の前提条件（クロスチェック）

以下の重要なエリア間依存を検証し、全て妥当であることを確認:

| 対象ダンジョン | 前提 | 妥当性 |
|---------------|------|--------|
| C5-01 Spring Boot | C1-20 抽象化 | OK: DIにはインタフェースの理解が必須 |
| C4-04 JPA/Hibernate | C1-20 抽象化 + C4-03 JDBC | OK: ORMにはOOP+DB接続の両方が必要 |
| C1-22 コレクション | C1-13 配列 + C1-21 例外 | OK: 配列の発展形+例外処理の知識が必要 |
| C1-24 ラムダ式 | C1-23 ジェネリクス + C1-20 抽象化 | OK: 関数型IFにはジェネリクスと抽象化が前提 |
| C2-04 パフォーマンス | C2-03 メモリ + C1-26 並行処理 | OK: プロファイリングには両方の知識が必要 |
| C5-06 Spring Bootテスト | C5-02 MVC + C3-03 Mockito | OK: 統合テストにはモック技法が必須 |
| C6-05 分散システム | C6-03 K8s + C6-04 Kafka + C2-09 CA | OK: 分散設計の総合前提として妥当 |

### 設計判断の妥当性確認

**C1-09（switch）の前提がC1-07（if/else）のみ:**
switchは値のパターンマッチングであり、比較・論理演算子（C1-08）の知識は必須ではない。C1-07でif文の概念を理解していれば十分。妥当。

**C1-13（配列）の前提がC1-10（for文）のみ:**
配列の基本操作にはforループがあれば十分であり、ネスト・break・continue（C1-12）は不要。妥当。

## 補足: 設計上の良い点

1. **単一ルート**: C1-01が唯一の開始点であり、全学習者が同じ基盤から始まる
2. **適度な並行性**: レイヤー16以降で並行攻略可能なダンジョンが増え、学習者の興味に応じた柔軟なルート選択が可能
3. **エリア間依存の妥当性**: Spring->OOP、JPA->DB+OOP等のクロスカテゴリ依存が実務上の前提知識と整合
4. **職業割り当ての網羅性**: 65ダンジョンが基本職10職+C6に漏れなく・重複なく分配されている
5. **勇者到達の収束性**: 全パスが最終的にC6-05（分散システム設計）に収束し、勇者への明確なゴールが存在する
